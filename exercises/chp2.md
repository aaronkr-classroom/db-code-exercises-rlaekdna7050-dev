# 2장 연습문제

## 개념퍼즐

**가로**


. 복잡한 현실 세계의 데이터를 좀 더 쉽게 이해할 수 있도록 기술하는 개념적 도구 : 데이터모델

. 개념적 모델링 결과인 개념적 구조를 DBMS가 이해할 수 있는 논리적 구조, 즉 데이터 모델로 변환하는 과정 : 논리적모델링

. 디스크에 데이터가 저장될 수 있도록 논리적 구조를 물리적 구조로 변환하는 과정 : 물리적모델링

. 하나의 논리적 기능을 수행하기 위한 작업 단위 : 트랜잭션

. DB 설계 시 고려 사항 가운데 하나로, DB에 저장된 데이터 값은 제약조건을 항상 만족해야 한다는 것 : 무결성

**새로**

. 현실 세계에서 인식한 데이터를 추상화를 통해 개념 세계의 추상적 개념으로 표현하는 과정 : 개념적모델링

. 현실 세계의 데이터를 DB에 표현하기 위한 모델을 만드는 과정 : 데이터모델링

. DB 설계 시 고려 사항 가운데 하나로, 시스템 운영에 영향을 주지 않고 새로운 데이터나 응용 프로그램을 추가하는 것이가능해야 한다는 것 : 확장성

## 연습문제

1. (______) 안에 알맞은 용어는?
현실 세계의 데이터는 사람의 오감으로 인지할 수 있는 실체로 둘 이상의 특성으로 구성된 (개체)로표현된다.

2. (______) 안에 알맞은 용어는?
개넘 세계의 데이터는 개체의 의미로부터 인식된 개념으로， 둘 이상의 속성으로 구성된 (개체 타입)으로표현된다.

3. 현실 세계의 직장인이 속한 세계에 따라 구분되는 구성 요소를 5개 이상 작성하시오.

| 구분 | 구성요소 |
|-----|----------|
| 현실 세계의 직장인 | 이름, 성별, 최종학력, 전공, 직장명, 취미, 특기, 가족관계, 휴대폰번호 등 |
| 회사라는 개념 세계의 직장인 | 이름, 사원번호, 주민등록번호, 근무부서, 담당업무, 연봉, 입사일자 등 |
| 자동차 보험사라는 개념 세계의 직장인 | 이름, 주민등록번호, 직업, 차량번호, 배기량, 사고이력, 마일리지 등 |
| 회사라는 개념 세계와 대응하는 컴퓨터 세계의 직장인 | 이름(문자 10자리), 사원번호(문자 5자리), 주민등록번호(문자 13자리), 근무부서(숫자 3자리), 담당업무(문자 20자리), 연봉(숫자 10자리), 입사일자(DATE) 등 |

4. 데이터 모델령의 각단계와설명을바르게 연결하시오.
   
  1. 개념적스카마모텔링 : 현실 세계에서 인석한 데이터를 추상화를 통해서 개념 세계의 추상적 개념으로 표현하는 과정
  2. 논리적스키마모델링 : 개념적 모델링 결과인 개념적 구조를 DBMS가이해할 수 있는 논리적 구조， 즉 데이터 모델로 변환하는과정
  3. 물리적스키마모델렁 : 디스크에 데이터가 저장될 수 있도록 논리적 구조를 물리적 구조로 변환하는 과정

5. 데이터 모텔링을 개념적 모댈링， 논리적 모댈링， 물리적 모댈링으로 구분해서 수행히는 이유를 설명하시오.

처음부터 디스크와 같은 물리적 저장장치에 저장된 DB를 구축하는 것이 매우 어려운
작업이므로, 현실 세계의 데이터를 사람이 이해하기 쉬운 개념적 데이터 모델로 표현한
후, DBMS가 이해할 수 있는 논리적 데이터 모델로 변환한 다음, 실제로 디스크에 저장
할 수 있는 물리적 구조로 변환하는 것이 효율적이다.

6. 개념적 구조， 즉 개념적 모댈링 결과를 DBMS가 직접 이해할수 없는 이유를 설명하시오.

개념적 모델링 결과는 주로 ER 다이어그램으로 표현되는데, 이것은 사람들 즉, DB 설계
자의 이해를 돕기 위한 개념적 구조이므로 DBMS는 직접 이해할 수 없다. 따라서 개념
적 구조를 DBMS가 이해할 수 있는 논리적 데이터 모델로 변환해야 한다.

7. 데이터 모델을구성히는세 가지 구성 요소를쓰시오.

데이터 구조((data structure), 허용되는 연산(operation), 제약조건(constraints)

8. 개념적 데이터 모벨의 구성 요소를 쓰시오.

  개체, 관계

9. 다음 중 데이터베이스를 구축하는 6단계를 바르게 나열한 것은? 4
 
  1. 요구 수집 및 분석 - DBMS 선정 - 개념적 설계 - 논리적 설계 - 물리적 설계 - 구현 및 테스트
  2. 요구 수집 및 분석 - DBMS 선정 - 논리적 설계 - 개념적 설계 - 물리적 설계 - 구현 및 태스트
  3. 요구 수집 및 분석 - 개념적 설계 - DBMS 선정 - 논리적 설계 - 물리적 설계 - 구현 및 테스트
  4. 요구 수집 및 분석 - 개념적 설계 - 논리적 설계 - DBMS 선정 - 물리적 설계 - 구현 및 태스트

데이터베이스 구축은 요구 수집 및 분석, 개념적 설계, DBMS 선정, 논리적 설계, 물리적 설계, 그리고 구현 및 테스트라는 6단계로 진행한다. 요구 수집 및 분석 단계에
서는 사용자의 요구사항을 수집 및 분석해서 업무별 데이터 및 트랜잭션 요구 분석 명세서를 작성하고, 개념적 DB 설계 단계에서는 개념적 스키마 모델링과 트랜잭션 모델링을 수행한다. 
DBMS 선정 단계에서는 논리적 데이터 모델과 운영 환경, 경제성 등을 고려해서 적절한 DBMS 선정하고, 논리적 DB 설계 단계에서는 논리적 스키마 모델링과 트랜잭션 인터페이스 설계 등을 수행한다. 
물리적 DB 설계 단계에서는 물리적 스키마 모델링과 트랜잭션 상세 설계를 수행하고, 구현 및 테스트 단계에서는 DB 생성 및 초기데이터 삽입, 그리고 트랜잭션 처리용 응용 프로그램 작성 및 테스트를 수행한다.

10. 다음 중 DB 설계 전략에 대한 설명이 잘못된 것은? 3
 
  1. DB의 내용과 구조에 치중해서 설계히는 방법은 데이터 중심 설계다.
  2. 데이터의 처리와응용에 치중해서 설계하는 방법은 처리 중심 설계다.
  3. 데이터 중심 설계와 처리 중심 설계 가운데 하나를 선택해서 진행하는 것이 효율적이다.
  4. 데이터 중심 설계와 처리 중심 설계를 병행해서 진행히는 것이 효율적이다.

11. 다음 중 데이터베이스 설계 시 고려 사항에 대한 설명이 잘못된 것은? 2
 
  1. 보안: 불법적인 접근(데이터의 변경 ， 손실， 노출 등)에 대해서 DB를 보호할 수 있어야한다.
  2. 확장성:시스템운영을일시중단하고새로운데이터나응용프로그램을 추가할수있어야한다.
  3. 일관성: 저장된 데이터 값간또는특정 질의에 대한응답들 간에 상호 모순 없이 일치해야한다.
  4. 무결성 :DB에 저장된 데이터 값은제약조건을항상만족해야한다.
